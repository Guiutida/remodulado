# Plano 1: Instalar Dependências de Segurança

**Fase:** 1 — Segurança e Fundação  
**Objetivo:** Instalar os 6 pacotes npm de segurança necessários para os planos seguintes e adicionar as variáveis de ambiente JWT_SECRET e CORS_ORIGIN ao .env.  
**Requisitos cobertos:** AUTH-01, AUTH-02, AUTH-03, AUTH-05 (pré-requisito de todos)  
**Dependências:** Nenhuma — este é o primeiro plano  
**Estimativa:** 2 tarefas, ~30 minutos

---

## Contexto

Os planos 2–5 dependem de `bcrypt`, `jsonwebtoken`, `helmet`, `cors`, `express-rate-limit` e `express-validator`. Nenhum desses pacotes existe no `backend/package.json` ainda. Instalar tudo em lote agora evita interrupções nas tarefas de implementação.

O `.env` atual tem 6 variáveis de banco e porta. É preciso adicionar `JWT_SECRET` (segredo de assinatura JWT) e `CORS_ORIGIN` (origem permitida no CORS) antes de qualquer código que as consuma. Sem elas, o servidor lança erro ao iniciar após o PLAN-03.

**O que NÃO muda neste plano:** nenhum arquivo `.js` de produção é alterado. Este plano é puramente de infraestrutura.

---

## Tarefas

### Tarefa 1: Instalar pacotes npm de segurança

**Arquivo(s):** `backend\package.json`, `backend\package-lock.json`  
**Ação:** criar/instalar

Execute o seguinte comando a partir do diretório `backend\`:

```bash
cd backend
npm install bcrypt@6.0.0 jsonwebtoken@9.0.3 helmet@8.2.0 cors express-rate-limit@8.5.2 express-validator@7.3.2
```

> **Por que essas versões?** Fixadas conforme a Auditoria de Legitimidade do RESEARCH.md. `cors` não tem versão fixada — a `2.8.6` é estável. Todos os 6 pacotes foram verificados em npmjs.com contra repositórios oficiais conhecidos (Auth0, helmetjs, expressjs, etc.).

> **Não instalar:** `bcryptjs` (puro JS, 2–3× mais lento que `bcrypt` com bindings C++). Usar sempre `bcrypt` com bindings nativos.

Após a instalação, confirmar que as entradas aparecem em `dependencies` no `package.json`:

```json
"bcrypt": "^6.0.0",
"cors": "^2.8.6",
"express-rate-limit": "^8.5.2",
"express-validator": "^7.3.2",
"helmet": "^8.2.0",
"jsonwebtoken": "^9.0.3"
```

**Verificação:**
```bash
cd backend && node -e "require('bcrypt'); require('jsonwebtoken'); require('helmet'); require('cors'); require('express-rate-limit'); require('express-validator'); console.log('Todos os 6 pacotes instalados com sucesso')"
```

### Tarefa 2: Adicionar JWT_SECRET e CORS_ORIGIN ao .env

**Arquivo(s):** `backend\.env`  
**Ação:** editar

O `.env` atual contém:
```
DB_HOST=127.0.0.1
DB_USER=root
DB_PASSWORD=
DB_NAME=duopratic
DB_PORT=3307
PORT=3000
```

Adicionar ao final do arquivo (preservar as linhas existentes sem alteração):
```
JWT_SECRET=troque_este_valor_por_uma_string_longa_e_aleatoria_antes_de_usar
CORS_ORIGIN=http://localhost:3000
NODE_ENV=development
```

> **Regra crítica:** O valor de `JWT_SECRET` acima é um placeholder. Nunca commitar o `.env` com um segredo real. Confirmar que `backend\.env` consta no `.gitignore` — se não constar, adicionar agora.

> **Por que CORS_ORIGIN aqui?** O PLAN-03 substitui o CORS wildcard pelo pacote `cors` com `origin: process.env.CORS_ORIGIN`. Se a variável não existir no `.env` quando o servidor subir, o Express usa `http://localhost:3000` como fallback (hardcoded no PLAN-03), mas é boa prática ter a variável explícita desde já.

**Verificação:**
```bash
cd backend && node -e "require('dotenv').config(); console.log('JWT_SECRET:', process.env.JWT_SECRET ? 'definido' : 'AUSENTE'); console.log('CORS_ORIGIN:', process.env.CORS_ORIGIN || 'AUSENTE')"
```

## Ameaças e Mitigações

| Ameaça | Mitigação |
|--------|-----------|
| Instalar pacote malicioso com nome semelhante (typosquatting) | Todos os 6 pacotes verificados no RESEARCH.md via `npm view` contra organizações oficiais. Confirmar URLs: `npmjs.com/package/bcrypt`, `npmjs.com/package/jsonwebtoken`, etc. |
| JWT_SECRET fraco em produção | O placeholder deixado no `.env` é intencionalmente óbvio. Gerar valor forte antes de qualquer deploy: `node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"` |
| `.env` commitado com segredo | Verificar `.gitignore` neste plano. Se `backend\.env` não constar, adicionar antes de qualquer commit. |
