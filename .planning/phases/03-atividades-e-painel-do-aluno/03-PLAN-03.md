# Plan 03-03: API de Respostas e Feedback Imediato (Aluno)

**Phase:** 3 — Atividades e Painel do Aluno
**Goal:** Endpoints para o aluno submeter respostas (MC com feedback imediato, dissertativa como "aguardando correção") e consultar suas respostas — com streak e pontuação atualizados atomicamente.
**Requirements covered:** ATIV-04, ATIV-05
**Dependencies:** PLAN-01 (tabelas `questoes`, `respostas_questao`), PLAN-02 (`verificarAlunoNaAtividade` já no controller, rota montada)
**Estimated complexity:** High

---

## Contexto

Este PLAN **modifica** `backend/controllers/atividadesController.js` e
`backend/routes/atividades.js` criados no PLAN-02.

**Regra de negócio central:**
- Múltipla escolha: `correta = (resposta.toLowerCase() === gabarito.toLowerCase()) ? 1 : 0`
- Dissertativa: `correta = NULL` (Phase 4 corrige com IA)
- Submissão é **idempotente**: `ON DUPLICATE KEY UPDATE` em `respostas_questao`
- `gabarito_correto` só é enviado na resposta quando `tipo === "multipla_escolha"` E `correta === 0` (feedback de correção)
- Nunca enviar `gabarito` para dissertativa

**Pontuação:** +10 por entrega + +5 por acerto MC. Streak: incrementa se ontem acessou, reinicia se parou, mantém se já acessou hoje.

**Função `responderAtividade`:**

```js
async function responderAtividade(req, res) {
    try {
        const atividadeId = parseInt(req.params.id, 10);
        const alunoId     = req.usuario.id; // NUNCA do body

        if (!(await verificarAlunoNaAtividade(atividadeId, alunoId))) {
            return res.status(403).json({ status: "erro", message: "Acesso negado." });
        }

        const { respostas } = req.body;
        if (!Array.isArray(respostas) || respostas.length === 0) {
            return res.status(400).json({ status: "erro", message: "Array de respostas é obrigatório." });
        }

        const resultados = [];
        let acertosMC = 0;

        for (const item of respostas) {
            const questaoId = parseInt(item.questao_id, 10);
            const resposta  = String(item.resposta || "").trim();

            if (!questaoId || !resposta) continue;

            // Busca questão — inclui gabarito (uso interno do servidor, nunca retornado ao aluno)
            const [questoes] = await banco.query(
                "SELECT id, tipo, gabarito FROM questoes WHERE id = ? AND atividade_id = ? LIMIT 1",
                [questaoId, atividadeId]
            );
            if (!questoes.length) continue;

            const questao = questoes[0];
            let correta   = null; // default: dissertativa aguardando correção

            if (questao.tipo === "multipla_escolha") {
                correta = resposta.toLowerCase() === questao.gabarito.toLowerCase() ? 1 : 0;
                if (correta === 1) acertosMC++;
            }

            // Idempotente: re-submissão atualiza resposta e resultado
            await banco.query(
                `INSERT INTO respostas_questao (aluno_id, questao_id, resposta, correta)
                 VALUES (?, ?, ?, ?)
                 ON DUPLICATE KEY UPDATE
                     resposta      = VALUES(resposta),
                     correta       = VALUES(correta),
                     respondido_em = CURRENT_TIMESTAMP`,
                [alunoId, questaoId, resposta, correta]
            );

            // Monta resultado para retornar ao aluno
            const resultado = { questao_id: questaoId, correta };
            // gabarito_correto: só para MC errada — feedback de correção
            if (questao.tipo === "multipla_escolha" && correta === 0) {
                resultado.gabarito_correto = questao.gabarito;
            }
            resultados.push(resultado);
        }

        // Registra entrega (idempotente)
        await banco.query(
            `INSERT INTO entregas (atividade_id, aluno_id, status, enviado_em)
             VALUES (?, ?, 'entregue', NOW())
             ON DUPLICATE KEY UPDATE
                 status     = 'entregue',
                 enviado_em = NOW()`,
            [atividadeId, alunoId]
        );

        // Atualiza streak e pontuação
        // streak_atual:
        //   - se ultimo_acesso = hoje        → mantém (já estudou hoje)
        //   - se ultimo_acesso = ontem       → incrementa
        //   - qualquer outra data (ou NULL)  → reinicia em 1
        await banco.query(
            `UPDATE usuarios SET
                pontuacao    = pontuacao + 10 + (5 * ?),
                streak_atual = CASE
                    WHEN DATE(ultimo_acesso) = CURDATE()                                  THEN streak_atual
                    WHEN DATE(ultimo_acesso) = DATE_SUB(CURDATE(), INTERVAL 1 DAY)        THEN streak_atual + 1
                    ELSE 1
                END,
                ultimo_acesso = CURDATE()
             WHERE id = ?`,
            [acertosMC, alunoId]
        );

        res.json({ status: "ok", resultados });

    } catch (erro) {
        res.status(500).json({ status: "erro", message: "Erro ao registrar respostas.", detalhe: erro.message });
    }
}
```

**Função `getRespostasAluno`:**

```js
async function getRespostasAluno(req, res) {
    try {
        const atividadeId = parseInt(req.params.id, 10);
        const alunoId     = req.usuario.id;

        if (!(await verificarAlunoNaAtividade(atividadeId, alunoId))) {
            return res.status(403).json({ status: "erro", message: "Acesso negado." });
        }

        // Retorna respostas do aluno — SEM campo gabarito
        const [respostas] = await banco.query(
            `SELECT rq.questao_id, rq.resposta, rq.correta, q.tipo, rq.respondido_em
             FROM respostas_questao rq
             JOIN questoes q ON q.id = rq.questao_id
             WHERE rq.aluno_id = ? AND q.atividade_id = ?
             ORDER BY q.ordem ASC`,
            [alunoId, atividadeId]
        );

        res.json({ status: "ok", respostas });

    } catch (erro) {
        res.status(500).json({ status: "erro", message: "Erro ao buscar respostas.", detalhe: erro.message });
    }
}
```

## Verification

- [ ] `POST /api/atividades/:id/respostas` sem token → 401
- [ ] `POST /api/atividades/:id/respostas` com token de professor → 403
- [ ] `POST /api/atividades/:id/respostas` com token de aluno de outra turma → 403
- [ ] `POST /api/atividades/:id/respostas` com body `{ respostas: [{ questao_id: X, resposta: "b" }] }` (MC, gabarito "b") → 200, `resultados[0].correta === 1`, sem campo `gabarito_correto`
- [ ] `POST /api/atividades/:id/respostas` com resposta MC errada → `resultados[0].correta === 0`, `resultados[0].gabarito_correto` presente
- [ ] `POST /api/atividades/:id/respostas` com questão dissertativa → `resultados[0].correta === null`, sem campo `gabarito_correto`
- [ ] Re-submissão da mesma resposta → sem erro 500/409, retorna resultado atualizado (idempotente)
- [ ] Após submissão: banco `usuarios.pontuacao` aumentou; `usuarios.streak_atual` ≥ 1; `usuarios.ultimo_acesso` = data de hoje
- [ ] Após submissão: registro existe em `entregas` com `status = "entregue"`
- [ ] `GET /api/atividades/:id/respostas` → lista respostas sem campo `gabarito`
- [ ] `GET /api/atividades/:id/respostas` de aluno de outra turma → 403

## Security Checklist

- [ ] `alunoId` vem **exclusivamente** de `req.usuario.id` — nunca do `req.body`
- [ ] Campo `gabarito` nunca aparece em nenhum objeto retornado ao aluno
- [ ] `gabarito_correto` enviado **apenas** para questões MC com `correta === 0`
- [ ] Dissertativas nunca recebem `gabarito_correto` na resposta, mesmo que o campo exista no banco
- [ ] `questao_id` validado com `parseInt` e verificado contra `atividade_id` na query (evita injeção de questão de outra atividade)
- [ ] Resposta do aluno truncada/sanitizada como `String(item.resposta).trim()` antes de persistir
- [ ] Pontuação calculada no servidor — nunca aceita do body
