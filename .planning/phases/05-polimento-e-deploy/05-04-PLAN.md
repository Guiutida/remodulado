**Passos para criar o deploy na Railway (executar na ordem):**

**1. Criar conta e projeto Railway:**
- Acesse https://railway.app → Sign Up (recomendado via GitHub)
- No dashboard: New Project → Deploy from GitHub repo
- Selecione o repositório DuoPratic → confirmar

**2. Adicionar MySQL plugin:**
- No projeto Railway: clique em "Add Service" → "Database" → "MySQL"
- Aguarde o plugin provisionar (1-2 minutos)
- Clique no plugin MySQL → aba "Connect" → copie os valores de Host, Port, User, Password, Database

**3. Configurar variáveis de ambiente na aplicação Node.js:**
- Clique na aplicação Node.js (não no MySQL) → "Variables"
- Adicionar cada variável da lista (use os valores do MySQL plugin para os DB_*):
  - `JWT_SECRET` = gerar com: `node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"`
  - `GEMINI_API_KEY` = valor do seu backend/.env local
  - `DB_HOST`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`, `DB_PORT` = do painel MySQL plugin
  - `NODE_ENV` = `production`
  - `CORS_ORIGIN` = deixar vazio por agora (completar depois de obter a URL pública)

**4. Fazer deploy:**
- Railway detecta o repositório GitHub automaticamente
- O build usa `cd backend && npm install` (railway.json)
- Aguarde os logs de build — deve aparecer "Servidor DuoPratic online na porta ..."
- Se aparecer "❌ Variáveis de ambiente obrigatórias ausentes": verifique as vars do passo 3

**5. Obter URL pública e configurar CORS_ORIGIN:**
- Aplicação → Settings → Domains → "Generate Domain"
- Copie a URL (ex: `https://duopratic-production.railway.app`)
- Volte em Variables → adicionar `CORS_ORIGIN` = a URL copiada
- Railway faz redeploy automático com a nova var

**6. Rodar migration-05 contra o banco de produção:**
Com os valores de conexão do Railway MySQL em mãos, criar um arquivo `.env.prod` temporário (NÃO commitar):
```
DB_HOST=...  DB_USER=...  DB_PASSWORD=...  DB_NAME=...  DB_PORT=...
```
Depois executar:
```
node -e "
  process.env.DB_HOST='...';
  process.env.DB_USER='...';
  process.env.DB_PASSWORD='...';
  process.env.DB_NAME='...';
  process.env.DB_PORT='...';
" && node backend/scripts/migration-05.js
```
Ou simplesmente: editar temporariamente `backend/.env` com as vars de produção, rodar `node backend/scripts/migration-05.js`, depois restaurar o .env local.

**Confirmação esperada:** Railway dashboard mostra status "Active" (verde). Health check `/api/health` responde 200.
