## Spacing Scale

O projeto usa um grid de 8pt com exceções documentadas (observado em `assets/css/base.css`).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gaps de ícone, separadores finos |
| sm | 8px | Gaps compactos entre chips e filtros |
| md | 16px | Padding padrão de elementos de formulário |
| lg | 24px | Padding de painéis e seções |
| xl | 32px | Espaçamento de layout |
| 2xl | 48px | Quebras de seção maiores |
| 3xl | 64px | Espaçamento de página |

**Exceções estabelecidas (não alterar):**
- 6px — gaps internos ultra-compactos (`.marca`, dots de status)
- 14px — padding de nav (`.topo`) e labels de formulário
- 22px — padding de cartão padrão (`.cartao-painel`)
- 28px — padding de sidebar (`.barra-aluno`)
- 44px — target mínimo de toque para botões de ícone (`.icone-topo` 44×44px)

**Phase 5 — sem novos valores de espaçamento.** Usar exclusivamente os valores acima.

## Color

Todos os tokens vêm de `:root` em `base.css`. Phase 5 adiciona **apenas 2 variantes de toast** e **2 regras de erro inline** — nenhum token novo no `:root`.

| Role | Valor (light) | Valor (dark) | Usage |
|------|--------------|--------------|-------|
| Dominant (60%) | `var(--fundo)` = `#ffffff` | `#101827` | Background de página |
| Secondary (30%) | `var(--branco)` = `#fff` | `#172033` | Cards, sidebar, nav, modais |
| Accent (10%) | `var(--azul)` = `#3157f6` | `#3157f6` | Ver lista abaixo |
| Destructive | `#991b1b` | `#991b1b` | Toast de erro (`.aviso-config.erro`) |
| Success | `#166534` | `#166534` | Toast de sucesso (`.aviso-config.sucesso`) |

**Accent reservado estritamente para:**
- Botão `.botao` primário (background + border)
- `border-top-color` do `.spinner-anel`
- Borda `dashed` de `.area-upload`
- Estado `.ativo` nos itens de navegação lateral
- Etiquetas `.etiqueta` (background `#eef4ff`, text `var(--azul)`)

**Novas cores de toast (únicas adições):**

```css
/* Adicionar em base.css — seção após .aviso-config */
```

Dark mode: ambas as variantes mantêm as mesmas cores (contraste WCAG AA garantido em ambos os fundos).

## Component Contracts (Phase 5 Específicos)

### 1. Spinner Global

**Quem cria:** `ui.js → showLoading()` injeta uma única vez no `<body>` na primeira chamada; reutiliza nas subsequentes.

**Markup (gerado por JS, não hardcoded no HTML):**
```html
<div class="carregando-global" role="status" aria-label="Carregando" aria-live="polite">
  <div class="spinner-anel"></div>
  <span>Carregando…</span>
</div>
```

**CSS a adicionar em `base.css` (após `.aviso-config`):**
```css
.carregando-global {
  position: fixed;
  inset: 0;
  z-index: 50;
  display: none;
  flex-direction: column;
  align-items: center;
  justify-content: center;
  gap: 16px;
  background: rgba(15, 35, 68, 0.35);
}
.carregando-global.ativo { display: flex; }
.spinner-anel {
  width: 40px;
  height: 40px;
  border: 4px solid rgba(49, 87, 246, 0.2);
  border-top-color: var(--azul);
  border-radius: 50%;
  animation: girar 0.75s linear infinite;
}
@keyframes girar { to { transform: rotate(360deg); } }
.carregando-global span {
  color: #fff;
  font-weight: 700;
  font-size: 0.88rem;
  letter-spacing: 0.01em;
}
```

**Comportamento:**
- `showLoading()` → adiciona `.ativo` ao `.carregando-global`
- `hideLoading()` → remove `.ativo`
- **Nunca usar** em `ia-estudo.html` durante streaming SSE — essa página usa indicador de digitação próprio
- Sempre chamar `hideLoading()` no bloco `finally {}` do fetch para garantir que o spinner suma mesmo em caso de erro

### 2. Toast Notification (extensão do `.aviso-config` existente)

**CSS existente (base.css linhas 763–773) — não alterar:**
```css
.aviso-config {
  position: fixed;
  right: 24px;
  bottom: 24px;
  z-index: 30;
  border-radius: 12px;
  padding: 12px 16px;
  background: var(--texto);
  color: var(--branco);
  font-weight: 900;
  box-shadow: 0 14px 30px rgba(15, 35, 68, .22);
}
```

**Acrescentar apenas:**
```css
.aviso-config.sucesso { background: #166534; color: #f0fdf4; }
.aviso-config.erro    { background: #991b1b; color: #fef2f2; }
```

**Comportamento (API de `ui.js`):**
```js
showError(msg)    // → .aviso-config.erro, auto-dismiss 3000ms
showSuccess(msg)  // → .aviso-config.sucesso, auto-dismiss 3000ms
// Padrão neutro (avisos informativos): .aviso-config sem modificador, 3000ms
```

**Regras de comportamento:**
- Máximo 1 toast visível por vez — remover o anterior antes de inserir o novo
- Auto-dismiss: **3000ms** (uniformizar — atual varia entre 1800ms e 2200ms em acesso.js/aluno.js)
- Posição: canto inferior direito a 24px das bordas (já definido — não alterar)
- Não bloquear interação do usuário (não é modal)

### 3. Erro Inline (`.erro-inline`)

Já referenciado em `atividades.js:275` mas sem estilo definido. Usar em listas quando o fetch falha e não há dados para renderizar.

**CSS a adicionar:**

```css
.erro-inline {

  font-size: 0.88rem;

body.tema-escuro .erro-inline { color: #fca5a5; }
```

**Uso:**
```js

```

### 4. Botão em Estado de Carregamento

**Padrão já existente em `acesso.js` — formalizar para todas as páginas:**

```js
// Antes do fetch:
botao.disabled = true;
botao.textContent = "Salvando…"; // ou "Enviando…", "Carregando…", "Salvando foto…"

// No finally {}:
botao.disabled = false;
botao.textContent = textoOriginal; // salvar antes com: const textoOriginal = botao.textContent;
```

**Regras:**
- Restaurar SEMPRE no `finally {}`, nunca apenas no `catch`
- **Não** adicionar spinner inline ao botão — o `.carregando-global` cobre este caso
- Formulários de acesso (`acesso.js`) já têm o padrão implementado — **não reescrever**
- Target mínimo do botão: 44px height (já garantido por `.botao { padding: 12px 20px }`)

### 5. Componente de Upload de Foto de Perfil

O modal já está **completamente estilizado** em `base.css` (linhas 774–844). **Nenhum CSS novo** para o modal.

**Elementos já presentes e estilizados:**
- `.modal-foto` / `.janela-foto` — modal com backdrop
- `.area-upload` — drop zone com borda dashed azul
- `.area-upload.arrastando` — estado de drag-over
- `.area-corte` / `.preview-foto` — preview com crop circular
- `.input-foto { display: none }` — input file oculto
- `.avatar-mini.com-foto` / `.avatar-aluno.com-foto` — estados com foto

**Comportamento novo (Plan 1 — multer backend):**
- Upload via `FormData` → `fetch POST /api/perfil/foto` (multipart/form-data, **sem** `Content-Type` header para FormData)
- Durante upload: `showLoading()` → ao concluir: `hideLoading()` + `showSuccess("Foto atualizada!")`
- Em erro: `hideLoading()` + `showError("Não foi possível salvar a foto. Tente novamente.")`
- Avatar renderiza `<img src="/uploads/{filename}">` em vez de `src="data:image/..."` base64
- Máximo: aceitar apenas `image/jpeg`, `image/png`, `image/webp` (validar no `<input accept="...">` e no backend)
- Sem novo CSS — usar `.com-foto` existente para ocultar iniciais quando `<img>` carrega

**Acessibilidade do avatar:**
```html
<img src="/uploads/foto.jpg" alt="Foto de perfil de {nome}" width="52" height="52">
```

## Copywriting Contract

| Elemento | Texto |
|---------|------|
| Spinner label | "Carregando…" |
| Toast sucesso — salvar config | "Salvo com sucesso" |
| Toast sucesso — upload foto | "Foto atualizada!" |
| Toast erro — genérico | "Algo deu errado. Tente novamente." |
| Toast erro — rede/offline | "Sem conexão com o servidor. Verifique sua internet." |
| Toast erro — sessão expirada | "Sua sessão expirou. Faça login novamente." |
| Botão loading — salvar | "Salvando…" |
| Botão loading — enviar formulário | "Enviando…" |
| Botão loading — upload foto | "Salvando foto…" |
| Botão loading — carregando dados | "Carregando…" |
| Erro inline — lista sem dados | "Não foi possível carregar. Tente novamente." |
| Upload foto — drag ativo | "Solte aqui" |
| Ação destrutiva | Nenhuma ação destrutiva nova nesta fase |

**Regras de tom:**
- Todos os textos em pt-BR, sem maiúsculas desnecessárias
- Nunca expor stack trace ou mensagem técnica da API ao usuário
- Mensagens de erro descrevem **o que aconteceu** + **o que fazer** (ex: "Sem conexão… Verifique sua internet")
- "…" com reticências tipográficas (U+2026) nos estados de loading, não três pontos separados
