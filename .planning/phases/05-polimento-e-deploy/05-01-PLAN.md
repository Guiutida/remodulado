<objective>
Migrar foto de perfil de LONGTEXT base64 para sistema de arquivos local via multer@2.1.1.

Purpose: Base64 em banco polui queries, aumenta payload em centenas de KB por request e é anti-pattern consolidado. A foto passa a ser um arquivo servido estaticamente.
Output: Endpoint POST /api/usuarios/perfil/foto com multer; migration-05.js que limpa base64 existentes; aluno.js com upload via FormData; diretório uploads/.
</objective>

```
const multer = require('multer');
const pathMod = require('path');
const fs = require('fs');

const UPLOADS_DIR = pathMod.join(__dirname, '..', '..', 'uploads');
fs.mkdirSync(UPLOADS_DIR, { recursive: true });

const storage = multer.diskStorage({
  destination: function (req, file, cb) { cb(null, UPLOADS_DIR); },
  filename: function (req, file, cb) {
    const ext = pathMod.extname(file.originalname).toLowerCase() || '.jpg';
    cb(null, Date.now() + '-' + Math.round(Math.random() * 1e9) + ext);
  }
});

const uploadFoto = multer({
  storage,
  limits: { fileSize: 5 * 1024 * 1024 },
  fileFilter: function (req, file, cb) {
    const permitidos = ['image/jpeg', 'image/png', 'image/webp'];
    if (permitidos.includes(file.mimetype)) {
      cb(null, true);
    } else {
      cb(new multer.MulterError('LIMIT_UNEXPECTED_FILE', 'Tipo de arquivo não permitido. Use JPG, PNG ou WebP.'));
    }
  }
});
```

<threat_model>
## Trust Boundaries

| Boundary | Descrição |
|----------|-----------|
| Browser → POST /api/usuarios/perfil/foto | Multipart file upload cruza aqui; payload não confiável |
| uploads/ → express.static (Plan 02) | Arquivos servidos como estáticos; nomes gerados pelo servidor, não pelo cliente |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | multer fileFilter | mitigate | `fileFilter` valida `file.mimetype` (busboy detecta MIME real, não extensão); tipos permitidos: jpeg/png/webp |
| T-05-02 | Denial of Service | multer limits | mitigate | `limits: { fileSize: 5 * 1024 * 1024 }` — rejeita uploads > 5 MB |
| T-05-03 | Elevation of Privilege | Nome do arquivo em uploads/ | mitigate | Nomes gerados por `Date.now() + random` — sem dados do usuário no nome; path não aceitável como input do cliente |
| T-05-04 | Spoofing | autenticar middleware | mitigate | `autenticar` já verifica JWT antes de multer processar o arquivo; sem autenticação → 401 antes do upload |
| T-05-SC | Tampering | npm install multer@2.1.1 | mitigate | multer verificado via github.com/expressjs/multer (organização oficial); versão 2.1.1 corrige CVEs de 2025/2026 da v1.x |
</threat_model>
