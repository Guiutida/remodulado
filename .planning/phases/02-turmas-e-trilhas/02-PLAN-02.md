# PLAN-02: API de Turmas Completa

## Goal
Implementar todos os endpoints de turmas (criação com código único, listagem, membros, remoção, entrada pelo código e consulta de turma do aluno), registrá-los no servidor e protegê-los com JWT + guards de perfil + IDOR checks.

## Requirements Covered
- TURM-01 (POST /api/turmas — professor cria turma com nome e código gerado no backend)
- TURM-02 (POST /api/turmas/entrar — aluno entra pelo código de 6 chars)
- TURM-03 (GET /api/turmas + GET /api/turmas/:id/membros — professor lista turmas e membros)
- TURM-04 (DELETE /api/turmas/:id/membros/:alunoId — professor remove aluno)
- TURM-05 (GET /api/aluno/turma — aluno consulta sua turma atual)

## Pre-conditions
- PLAN-01 concluído: tabelas `trilhas`, `trilha_etapas`, `progresso_etapa` existem (não usadas aqui, mas a migration foi rodada e o banco está estável)
- `backend/middleware/auth.js` exporta `{ autenticar }` (já existe)
- `backend/db.js` exporta pool `mysql2/promise` (já existe)
- Tabelas `turmas` e `turma_alunos` existem em `database/schema.sql` (já existem desde Phase 1)

## Tasks

### Tarefa 1 — Criar `backend/controllers/turmasController.js`

Criar o arquivo com as 6 funções exportadas. Seguir **exatamente** o padrão de `backend/controllers/usuariosController.js`: `const banco = require("../db")`, handlers async com `try/catch`, resposta `{ status: "ok"|"erro", <recurso>: ... }`.

**Constante de charset e helper de código único** (no topo do arquivo, antes das funções):
```js
const CHARS = "ABCDEFGHJKLMNPQRSTUVWXYZ23456789"; // sem ambíguos: 0/O, 1/I

async function gerarCodigoUnico() {
    for (let tentativa = 0; tentativa < 10; tentativa++) {
        let codigo = "";
        for (let i = 0; i < 6; i++) codigo += CHARS[Math.floor(Math.random() * CHARS.length)];
        const [existente] = await banco.query(
            "SELECT id FROM turmas WHERE codigo = ? LIMIT 1", [codigo]
        );
        if (!existente.length) return codigo;
    }
    throw new Error("Não foi possível gerar código único após 10 tentativas.");
}
```

**Helper IDOR** (depois do helper de código):
```js
async function verificarDonoDaTurma(turmaId, professorId) {
    const [turmas] = await banco.query(
        "SELECT id FROM turmas WHERE id = ? AND professor_id = ? LIMIT 1",
        [turmaId, professorId]
    );
    return turmas.length > 0;
}
```

**Função `criar`** — POST /api/turmas (professor):
- Validar `nome` e `disciplina` no body; retornar 400 se ausentes
- Chamar `gerarCodigoUnico()`
- `INSERT INTO turmas (nome, disciplina, codigo, professor_id) VALUES (?, ?, ?, ?)`
- Retornar `201` com `{ status: "ok", turma: { id: resultado.insertId, nome, disciplina, codigo } }`
- Capturar `ER_DUP_ENTRY` (race condition rara) → 409

**Função `listar`** — GET /api/turmas (professor):
- Query: `SELECT t.id, t.nome, t.disciplina, t.codigo, t.criado_em, COUNT(ta.id) AS total_alunos FROM turmas t LEFT JOIN turma_alunos ta ON ta.turma_id = t.id WHERE t.professor_id = ? GROUP BY t.id ORDER BY t.criado_em DESC`
- Parâmetro: `[req.usuario.id]`
- Retornar `{ status: "ok", turmas: [...] }`

**Função `getMembros`** — GET /api/turmas/:id/membros (professor):
- `parseInt(req.params.id, 10)` para `turmaId`
- IDOR: `verificarDonoDaTurma(turmaId, req.usuario.id)` → se `false`, retornar 403
- Query: `SELECT u.id, u.nome, u.email, ta.criado_em AS entrou_em FROM turma_alunos ta JOIN usuarios u ON u.id = ta.aluno_id WHERE ta.turma_id = ? ORDER BY ta.criado_em ASC`
- Retornar `{ status: "ok", membros: [...] }`

**Função `removerMembro`** — DELETE /api/turmas/:id/membros/:alunoId (professor):
- `parseInt` em ambos os params
- IDOR: `verificarDonoDaTurma(turmaId, req.usuario.id)` → 403 se falhar
- `DELETE FROM turma_alunos WHERE turma_id = ? AND aluno_id = ?`
- Se `resultado.affectedRows === 0` → 404 "Aluno não encontrado nesta turma."
- Retornar `{ status: "ok", message: "Aluno removido da turma." }`

**Função `entrar`** — POST /api/turmas/entrar (aluno):
- Validar `codigo` no body; retornar 400 se ausente
- Verificar se aluno já está em alguma turma: `SELECT id FROM turma_alunos WHERE aluno_id = ? LIMIT 1` → se existir, retornar 409 "Você já está em uma turma. Saia da turma atual para entrar em outra."
- Buscar turma pelo código: `SELECT id, nome, disciplina FROM turmas WHERE codigo = ? LIMIT 1` com `[codigo.trim().toUpperCase()]` → 404 se não encontrar
- `INSERT INTO turma_alunos (turma_id, aluno_id) VALUES (?, ?)`
- Capturar `ER_DUP_ENTRY` → 409 "Você já é membro desta turma." (fallback de race condition)
- Retornar `201` com `{ status: "ok", turma: { id, nome, disciplina } }`

**Função `getTurmaDoAluno`** — GET /api/aluno/turma (aluno):
- Query JOIN: `SELECT t.id, t.nome, t.disciplina, t.codigo, u.nome AS professor_nome, (SELECT COUNT(*) FROM turma_alunos ta2 WHERE ta2.turma_id = t.id) AS total_membros FROM turma_alunos ta JOIN turmas t ON t.id = ta.turma_id JOIN usuarios u ON u.id = t.professor_id WHERE ta.aluno_id = ? ORDER BY ta.criado_em DESC LIMIT 1`
- Parâmetro: `[req.usuario.id]` (NUNCA `req.body.alunoId` — IDOR)
- Se nenhum resultado → 404 "Você não está em nenhuma turma."
- Retornar `{ status: "ok", turma: turmas[0] }`

**Exportar** no final do arquivo:

### Tarefa 2 — Criar `backend/routes/turmas.js`; modificar `backend/routes/alunos.js`; registrar em `backend/server.js`

**Criar `backend/routes/turmas.js`:**

Seguir padrão de `backend/routes/usuarios.js`. Guards de perfil inline (não extrair para arquivo compartilhado — padrão atual é inline por arquivo de rota):

```js
const { Router } = require("express");
const rateLimit = require("express-rate-limit");
const { autenticar } = require("../middleware/auth");
const turmasController = require("../controllers/turmasController");

const roteador = Router();

function verificarProfessor(req, res, next) {
    if (req.usuario.perfil !== "professor") {
        return res.status(403).json({ status: "erro", message: "Apenas professores podem realizar esta ação." });
    }
    next();
}

function verificarAluno(req, res, next) {
    if (req.usuario.perfil !== "aluno") {
        return res.status(403).json({ status: "erro", message: "Apenas alunos podem realizar esta ação." });
    }
    next();
}

const limitadorEntrada = rateLimit({
    windowMs: 15 * 60 * 1000,
    max: 20,
    standardHeaders: true,
    legacyHeaders: false,
    message: { status: "erro", message: "Muitas tentativas. Tente novamente em 15 minutos." }
});

// ORDEM IMPORTA: rotas literais antes de rotas com :param
roteador.post("/entrar", limitadorEntrada, autenticar, verificarAluno, turmasController.entrar);
roteador.post("/", autenticar, verificarProfessor, turmasController.criar);
roteador.get("/", autenticar, verificarProfessor, turmasController.listar);
roteador.get("/:id/membros", autenticar, verificarProfessor, turmasController.getMembros);
roteador.delete("/:id/membros/:alunoId", autenticar, verificarProfessor, turmasController.removerMembro);

module.exports = roteador;
```

## Verification

1. `GET /api/health` retorna 200 (servidor não quebrou)
2. Sem token: `POST /api/turmas` → 401
3. Com token de aluno: `POST /api/turmas` → 403 "Apenas professores..."
4. Com token de professor: `POST /api/turmas` com `{ "nome": "Turma Teste", "disciplina": "Matemática" }` → 201 com `codigo` de 6 chars (apenas A-Z sem O/I, 2-9 sem 0/1)
5. Com token de professor: `GET /api/turmas` → 200 com array incluindo a turma criada e `total_alunos: 0`
6. Com token de aluno: `POST /api/turmas/entrar` com `{ "codigo": "<codigo_criado>" }` → 201 com `turma.nome`
7. Segunda tentativa do mesmo aluno: `POST /api/turmas/entrar` → 409 "Você já está em uma turma..."
8. Com token de professor: `GET /api/turmas/:id/membros` → 200 com o aluno que entrou
9. Com token de professor: `DELETE /api/turmas/:id/membros/:alunoId` → 200 "Aluno removido da turma."
10. Com token de aluno (após reingressar): `GET /api/aluno/turma` → 200 com `turma.professor_nome`
11. IDOR check: professor B tenta `GET /api/turmas/:idDaTurmaDoA/membros` → 403

## Threat Model

| Ameaça STRIDE | Componente | Disposição | Mitigação |
|---------------|------------|------------|-----------|
| Spoofing | JWT nas rotas de turma | mitigar | Middleware `autenticar` em todas as rotas; token verificado com `JWT_SECRET`; 401 se ausente ou expirado |
| Tampering / IDOR | Professor modifica turma de outro professor | mitigar | `verificarDonoDaTurma(turmaId, req.usuario.id)` antes de todo GET/DELETE de turma específica; retorna 403 se professor_id não corresponde |
| Tampering / IDOR | Aluno marca remoção de si mesmo via DELETE direto | mitigar | DELETE /api/turmas/:id/membros/:alunoId exige perfil professor (guard `verificarProfessor`); aluno recebe 403 |
| Tampering / IDOR | Aluno usa `req.body.alunoId` para entrar como outro | mitigar | `entrar` usa apenas `req.usuario.id` (JWT); `req.body` fornece apenas o código da turma, nunca o ID do aluno |
| Information Disclosure | Professor vê alunos de turma alheia | mitigar | `getMembros` verifica `professor_id = req.usuario.id` antes do JOIN |
| Elevation of Privilege | Aluno cria turma | mitigar | Guard `verificarProfessor` em `POST /api/turmas`; retorna 403 |
| Denial of Service | Brute force de códigos de 6 chars em `/entrar` | mitigar | `express-rate-limit` aplicado ao `POST /entrar`: 20 tentativas / 15 min por IP |
| Denial of Service | Colisão de código de acesso no `gerarCodigoUnico` | aceitar | Loop de 10 tentativas; probabilidade de 10 colisões consecutivas é ~10^-60 para base de dados TCC |
