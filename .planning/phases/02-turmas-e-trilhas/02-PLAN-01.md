# PLAN-01: Migration SQL — Tabelas de Trilhas e Progresso

## Goal
Criar as três tabelas que não existem no banco (`trilhas`, `trilha_etapas`, `progresso_etapa`) via script SQL idempotente e script Node.js de execução, seguindo o padrão já estabelecido em `database/schema.sql` e `backend/scripts/migrar-senhas.js`.

## Requirements Covered
- TRIL-01 (tabela `trilhas` para criar trilha com nome/descrição/disciplina)
- TRIL-02 (tabela `trilha_etapas` com campo `tipo` ENUM para texto/vídeo/link)
- TRIL-03 (coluna `turma_id` em `trilhas` faz a atribuição professor→turma)
- TRIL-06 (tabela `progresso_etapa` para persistir progresso do aluno)

## Pre-conditions
- Phase 1 concluída: `database/schema.sql` aplicado no banco (tabelas `usuarios`, `turmas`, `turma_alunos`, `atividades`, `entregas`, `avisos`, `preferencias_usuario` existem)
- `backend/.env` com `DB_HOST`, `DB_USER`, `DB_PASS`, `DB_NAME=duopratic` configurados
- `backend/db.js` exporta pool `mysql2/promise` (já existe)

## Tasks

### Tarefa 1 — Criar `database/migration-02-trilhas.sql`

Criar o arquivo `database/migration-02-trilhas.sql` com o seguinte conteúdo exato:

```sql
-- Migration 02: Trilhas e Progresso
-- Executar UMA VEZ após database/schema.sql estar aplicado
-- Seguro para re-execução: CREATE TABLE IF NOT EXISTS

USE duopratic;

-- Trilhas criadas pelo professor e atribuídas a uma turma
CREATE TABLE IF NOT EXISTS trilhas (
    id           INT          AUTO_INCREMENT PRIMARY KEY,
    turma_id     INT          NOT NULL,
    professor_id INT          NOT NULL,
    titulo       VARCHAR(140) NOT NULL,
    disciplina   VARCHAR(80)  NOT NULL,
    descricao    TEXT,
    ativa        TINYINT(1)   NOT NULL DEFAULT 1,
    criado_em    TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (turma_id)     REFERENCES turmas(id)    ON DELETE CASCADE,
    FOREIGN KEY (professor_id) REFERENCES usuarios(id)  ON DELETE CASCADE
) ENGINE=InnoDB;

-- Etapas da trilha ordenadas; tipo determina como renderizar o conteúdo
CREATE TABLE IF NOT EXISTS trilha_etapas (
    id        INT          AUTO_INCREMENT PRIMARY KEY,
    trilha_id INT          NOT NULL,
    ordem     SMALLINT     NOT NULL,
    titulo    VARCHAR(140) NOT NULL,
    tipo      ENUM('texto','video','link') NOT NULL DEFAULT 'texto',
    conteudo  TEXT         NOT NULL,
    criado_em TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (trilha_id) REFERENCES trilhas(id) ON DELETE CASCADE,
    UNIQUE KEY etapa_unica (trilha_id, ordem)
) ENGINE=InnoDB;

-- Registro de progresso individual: um registro por aluno por etapa
CREATE TABLE IF NOT EXISTS progresso_etapa (
    id           INT        AUTO_INCREMENT PRIMARY KEY,
    aluno_id     INT        NOT NULL,
    etapa_id     INT        NOT NULL,
    concluido    TINYINT(1) NOT NULL DEFAULT 0,
    concluido_em TIMESTAMP  NULL,
    UNIQUE KEY progresso_unico (aluno_id, etapa_id),
    FOREIGN KEY (aluno_id) REFERENCES usuarios(id)      ON DELETE CASCADE,
    FOREIGN KEY (etapa_id) REFERENCES trilha_etapas(id) ON DELETE CASCADE
) ENGINE=InnoDB;
```

Notas de design (preservar como comentários no arquivo):
- `professor_id` em `trilhas` é desnormalização proposital: evita JOIN `trilhas→turmas→professor_id` em cada IDOR check
- `UNIQUE KEY etapa_unica (trilha_id, ordem)` impede duas etapas com mesma posição; reordenação não está no escopo da Fase 2
- `progresso_etapa.concluido` é simples 0/1; o timestamp `concluido_em` permite auditoria futura
- `NULLIF(COUNT(te.id), 0)` é necessário no cálculo de percentual para evitar divisão por zero em trilhas sem etapas

### Tarefa 2 — Criar `backend/scripts/migration-02.js`

Criar o arquivo `backend/scripts/migration-02.js` seguindo **exatamente** o padrão de `backend/scripts/migrar-senhas.js`:
- `require("dotenv").config({ path: require("path").join(__dirname, "../.env") })`
- `const banco = require("../db")`
- Função async com `try/catch`, `process.exit(0)` no sucesso, `process.exit(1)` no erro
- Executar cada `CREATE TABLE IF NOT EXISTS` como query separada via `await banco.query(sql)` — não concatenar em uma única string multi-statement
- Logar nome de cada tabela criada/verificada
- Estrutura final:

```js
// backend/scripts/migration-02.js
// Executar UMA ÚNICA VEZ: node backend/scripts/migration-02.js
// Cria as tabelas trilhas, trilha_etapas e progresso_etapa.

require("dotenv").config({ path: require("path").join(__dirname, "../.env") });

const banco = require("../db");

const TABELAS = [
    {
        nome: "trilhas",
        sql: `CREATE TABLE IF NOT EXISTS trilhas (
            id           INT          AUTO_INCREMENT PRIMARY KEY,
            turma_id     INT          NOT NULL,
            professor_id INT          NOT NULL,
            titulo       VARCHAR(140) NOT NULL,
            disciplina   VARCHAR(80)  NOT NULL,
            descricao    TEXT,
            ativa        TINYINT(1)   NOT NULL DEFAULT 1,
            criado_em    TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
            FOREIGN KEY (turma_id)     REFERENCES turmas(id)    ON DELETE CASCADE,
            FOREIGN KEY (professor_id) REFERENCES usuarios(id)  ON DELETE CASCADE
        ) ENGINE=InnoDB`
    },
    {
        nome: "trilha_etapas",
        sql: `CREATE TABLE IF NOT EXISTS trilha_etapas (
            id        INT          AUTO_INCREMENT PRIMARY KEY,
            trilha_id INT          NOT NULL,
            ordem     SMALLINT     NOT NULL,
            titulo    VARCHAR(140) NOT NULL,
            tipo      ENUM('texto','video','link') NOT NULL DEFAULT 'texto',
            conteudo  TEXT         NOT NULL,
            criado_em TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
            FOREIGN KEY (trilha_id) REFERENCES trilhas(id) ON DELETE CASCADE,
            UNIQUE KEY etapa_unica (trilha_id, ordem)
        ) ENGINE=InnoDB`
    },
    {
        nome: "progresso_etapa",
        sql: `CREATE TABLE IF NOT EXISTS progresso_etapa (
            id           INT        AUTO_INCREMENT PRIMARY KEY,
            aluno_id     INT        NOT NULL,
            etapa_id     INT        NOT NULL,
            concluido    TINYINT(1) NOT NULL DEFAULT 0,
            concluido_em TIMESTAMP  NULL,
            UNIQUE KEY progresso_unico (aluno_id, etapa_id),
            FOREIGN KEY (aluno_id) REFERENCES usuarios(id)      ON DELETE CASCADE,
            FOREIGN KEY (etapa_id) REFERENCES trilha_etapas(id) ON DELETE CASCADE
        ) ENGINE=InnoDB`
    }
];

async function migrar() {
    for (const tabela of TABELAS) {
        await banco.query(tabela.sql);
        console.log(`  Tabela '${tabela.nome}' verificada/criada.`);
    }
    console.log("Migration-02 concluída com sucesso.");
    process.exit(0);
}

migrar().catch((erro) => {
    console.error("Erro na migration-02:", erro.message);
    process.exit(1);
});
```

## Threat Model

| Ameaça STRIDE | Componente | Disposição | Mitigação |
|---------------|------------|------------|-----------|
| Tampering | `migration-02.js` — script com privilégio DDL | mitigar | Script não é exposto via API; roda apenas em linha de comando com acesso ao servidor; não está em nenhuma rota Express |
| Denial of Service | Execução repetida do script | aceitar | `CREATE TABLE IF NOT EXISTS` torna re-execução inofensiva; sem risco de perda de dados |
| Elevation of Privilege | FK `ON DELETE CASCADE` em `progresso_etapa` | aceitar | CASCADE seguro para TCC: deleção de etapa limpa progresso orfão, comportamento correto |
| Information Disclosure | Schema exposto se script tiver saída em logs | aceitar | Logs são internos ao servidor; estrutura de tabelas não é dado sensível |
