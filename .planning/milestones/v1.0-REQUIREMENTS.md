## v1 Requirements — Final Status

### Segurança e Autenticação

- [x] **AUTH-01**: Senhas armazenadas com hash bcrypt — ✓ Phase 1
- [x] **AUTH-02**: Todas as rotas de API protegidas por middleware JWT — ✓ Phase 1
- [x] **AUTH-03**: Token JWT emitido no login e validado a cada requisição autenticada — ✓ Phase 1
- [x] **AUTH-04**: Sessão do usuário persiste entre recarregamentos do browser — ✓ Phase 1
- [x] **AUTH-05**: CORS configurado sem wildcard — apenas origens permitidas — ✓ Phase 1
- [x] **AUTH-06**: Aluno não consegue acessar dados de outro aluno (IDOR eliminado) — ✓ Phase 1

### Turmas

- [x] **TURM-01**: Professor pode criar uma turma com nome e código de acesso — ✓ Phase 2
- [x] **TURM-02**: Aluno pode entrar em uma turma pelo código de acesso — ✓ Phase 2
- [x] **TURM-03**: Professor visualiza lista de membros da turma — ✓ Phase 2
- [x] **TURM-04**: Professor pode remover aluno da turma — ✓ Phase 2
- [x] **TURM-05**: Aluno visualiza a turma em que está inscrito — ✓ Phase 2

### Trilhas de Aprendizado

- [x] **TRIL-01**: Professor pode criar uma trilha com nome, descrição e disciplina — ✓ Phase 2
- [x] **TRIL-02**: Professor pode adicionar etapas à trilha (texto, vídeo ou link) — ✓ Phase 2
- [x] **TRIL-03**: Professor atribui trilha a uma turma — ✓ Phase 2
- [x] **TRIL-04**: Aluno visualiza trilhas disponíveis na sua turma — ✓ Phase 2
- [x] **TRIL-05**: Aluno navega pelas etapas da trilha em sequência — ✓ Phase 2
- [x] **TRIL-06**: Progresso do aluno na trilha é persistido no banco de dados — ✓ Phase 2
- [x] **TRIL-07**: Aluno pode marcar etapa como concluída — ✓ Phase 2

### Atividades e Exercícios

- [x] **ATIV-01**: Professor pode criar atividade com questões de múltipla escolha — ✓ Phase 3
- [x] **ATIV-02**: Professor pode criar atividade com questões dissertativas — ✓ Phase 3
- [x] **ATIV-03**: Professor atribui atividade a uma turma com prazo — ✓ Phase 3
- [x] **ATIV-04**: Aluno responde atividade e recebe retorno imediato nas questões objetivas — ✓ Phase 3
- [x] **ATIV-05**: Resposta do aluno é salva no banco de dados — ✓ Phase 3
- [x] **ATIV-06**: Professor visualiza entregas dos alunos para cada atividade — ✓ Phase 3

### Painel do Aluno

- [x] **ALUN-01**: Aluno visualiza progresso real nas trilhas (não hardcoded) — ✓ Phase 3
- [x] **ALUN-02**: Aluno visualiza histórico de atividades realizadas — ✓ Phase 3
- [x] **ALUN-03**: Aluno visualiza pontuação e sequência de estudos (streak) — ✓ Phase 3

### Painel do Professor

- [x] **PROF-01**: Professor visualiza desempenho geral da turma por atividade — ✓ Phase 4
- [x] **PROF-02**: Professor visualiza progresso individual de cada aluno nas trilhas — ✓ Phase 4
- [x] **PROF-03**: Professor pode criar e gerenciar avisos para a turma — ✓ Phase 3

### IA — Tutor de Estudos

- [x] **IA-01**: Aluno pode enviar dúvida sobre exercício e receber orientação da IA — ✓ Phase 4
- [x] **IA-02**: IA explica o raciocínio necessário SEM fornecer a resposta direta — ✓ Phase 4
- [x] **IA-03**: Resposta da IA é transmitida em streaming (SSE) para feedback ao vivo — ✓ Phase 4
- [x] **IA-04**: Rate limiting aplicado no endpoint de IA (máx. 8 req/min por usuário) — ✓ Phase 4
- [x] **IA-05**: Histórico da conversa de IA é salvo e usado como contexto nas próximas mensagens — ✓ Phase 4

### IA — Geração de Trilhas

- [x] **IA-06**: Professor pode solicitar à IA que gere sugestão de trilha para um tema — ✓ Phase 4
- [x] **IA-07**: Trilha gerada pela IA pode ser editada antes de ser publicada — ✓ Phase 4

### IA — Avaliação de Desempenho

- [x] **IA-08**: IA gera resumo de pontos fortes e fracos do aluno com base em respostas — ✓ Phase 4
- [x] **IA-09**: Professor visualiza resumo de desempenho da turma gerado por IA — ✓ Phase 4

### Infraestrutura e Qualidade

- [x] **INFRA-01**: `server.js` dividido em módulos (`routes/`, `middleware/`, `controllers/`) — ✓ Phase 1
- [x] **INFRA-02**: Dados de seed removidos de handlers de produção — ✓ Phase 1
- [x] **INFRA-03**: `foto_perfil` migrada de LONGTEXT base64 para armazenamento em arquivo — ✓ Phase 5
- [x] **INFRA-04**: Variáveis de ambiente validadas na inicialização do servidor — ✓ Phase 5
- [ ] **INFRA-05**: Sistema implantado em ambiente de produção acessível — ⚠ Gap aceito (deploy removido do escopo; railway.json pronto para deploy futuro)
