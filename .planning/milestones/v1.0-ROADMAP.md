# Milestone v1.0: TCC

**Status:** ✅ SHIPPED 2026-05-26
**Phases:** 1–5
**Total Plans:** 23

## Overview

Plataforma educacional funcional de ponta a ponta para apresentação do TCC. Alunos navegam por trilhas, resolvem atividades com correção automática, acompanham progresso e contam com tutor de IA socrático. Professores gerenciam turmas, criam conteúdo e visualizam desempenho com análise por inteligência artificial.

## Phases

### Phase 1: Segurança e Fundação

**Goal:** O sistema não possui vulnerabilidades críticas conhecidas e o servidor está organizado em módulos — base segura para todas as fases seguintes.
**Depends on:** Nada (primeira fase)
**Plans:** 5 plans

Plans:
- [x] 01-01: Instalar dependências de segurança e configurar variáveis de ambiente
- [x] 01-02: Migrar senhas para bcrypt e emitir JWT no login
- [x] 01-03: Implementar middleware JWT e corrigir IDOR
- [x] 01-04: Refatorar server.js em módulos
- [x] 01-05: Adicionar guards de rota no frontend

**Success Criteria delivered:**
1. ✅ Senhas armazenadas como hash bcrypt — texto plano nunca gravado no banco
2. ✅ Qualquer rota de API retorna 401 sem token JWT válido
3. ✅ Aluno autenticado não consegue acessar dados de outro aluno (403)
4. ✅ `server.js` substituído por `routes/`, `middleware/`, `controllers/`
5. ✅ Dados de seed removidos de handlers de produção

## Milestone Summary

**Key Decisions:**

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| bcrypt + JWT para autenticação | Substituir texto plano e sessões frágeis | ✓ Implementado em Phase 1 |
| Express modular (routes/controllers/middleware) | Código sustentável para TCC e produto futuro | ✓ Implementado em Phase 1 |
| multer@2.1.1 DiskStorage para foto_perfil | Eliminar base64 LONGTEXT do banco | ✓ Implementado em Phase 5 |
| Gemini 2.5 Flash (@google/genai@2.6.0) | Free tier cobre demo do TCC | ✓ Implementado em Phase 4 |
| SSE para streaming das respostas da IA | Sem dependências extras | ✓ Implementado em Phase 4 |
| Vanilla HTML/CSS/JS (sem framework frontend) | Consistência com base existente | ✓ Mantido em todas as fases |
| ui.js com 4 globals (showLoading/hideLoading/showError/showSuccess) | Zero conflito com mostrarAviso de aluno.js | ✓ Implementado em Phase 5 |
| IA explica sem revelar respostas (defesa pedagógica) | Requisito pedagógico central do TCC | ✓ Implementado em Phase 4 |

**Known Gaps:**

- **INFRA-05**: Deploy em URL pública (Railway/Render) removido do escopo — sistema funciona localmente e está pronto para ser implantado manualmente se necessário.

**Technical Debt:**

- railway.json e .env.example já criados (commit e840ea4) para deploy futuro fácil
- Uploads em DiskStorage são efêmeros em ambiente Railway (Railway usa filesystem efêmero) — para produção real, migrar para S3 ou equivalente
