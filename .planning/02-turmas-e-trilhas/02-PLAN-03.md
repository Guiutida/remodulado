# PLAN-03: API de Trilhas Completa

## Goal
Implementar todos os endpoints de trilhas (criar com atribuição, listar por turma, adicionar etapas, listar trilhas do aluno com progresso, navegar etapas em sequência, marcar etapa concluída e consultar percentual), registrá-los no servidor e proteger cada rota com JWT + guard de perfil + IDOR checks.

## Requirements Covered
- TRIL-01 (POST /api/trilhas — professor cria trilha com nome, descrição, disciplina, já atribuída a turma)
- TRIL-02 (POST /api/trilhas/:id/etapas — professor adiciona etapa com tipo texto/video/link)
- TRIL-03 (turma_id no body de POST /api/trilhas faz a atribuição imediata à turma)
- TRIL-04 (GET /api/trilhas/minhas — aluno vê trilhas disponíveis na sua turma)
- TRIL-05 (GET /api/trilhas/:id/etapas — etapas em sequência por `ordem ASC`)
- TRIL-06 (GET /api/trilhas/:id/progresso — percentual calculado no backend com SUM(IF()))
- TRIL-07 (POST /api/trilhas/etapas/:etapaId/concluir — aluno marca etapa, persiste no banco)

## Pre-conditions
- PLAN-01 concluído: tabelas `trilhas`, `trilha_etapas`, `progresso_etapa` existem no banco
- PLAN-02 concluído: `backend/server.js` já importa `rotasTurmas`; padrão de guards confirmado
- `backend/middleware/auth.js` exporta `{ autenticar }` (já existe)
- `backend/db.js` exporta pool `mysql2/promise` (já existe)

const banco = require("../db");

async function verificarDonoDaTrilha(trilhaId, professorId) {

**Função `criar`** — POST /api/trilhas (professor):
- Validar `titulo`, `disciplina`, `turma_id` no body → 400 se ausentes
- IDOR: `SELECT id FROM turmas WHERE id = ? AND professor_id = ? LIMIT 1` com `[turma_id, req.usuario.id]` → 403 "Turma não encontrada ou acesso negado." se não encontrar
- `INSERT INTO trilhas (turma_id, professor_id, titulo, disciplina, descricao) VALUES (?, ?, ?, ?, ?)` com `[turma_id, req.usuario.id, titulo, disciplina, descricao || null]`
- Retornar `201` com `{ status: "ok", trilha: { id: resultado.insertId, titulo, disciplina, descricao: descricao || null, turma_id } }`

**Função `listarPorTurma`** — GET /api/trilhas/turma/:turmaId (professor):
- `parseInt(req.params.turmaId, 10)` para `turmaId`
- IDOR: `SELECT id FROM turmas WHERE id = ? AND professor_id = ? LIMIT 1` → 403 se falhar
- Query com COUNT de etapas: `SELECT t.id, t.titulo, t.disciplina, t.descricao, t.ativa, COUNT(te.id) AS total_etapas FROM trilhas t LEFT JOIN trilha_etapas te ON te.trilha_id = t.id WHERE t.turma_id = ? GROUP BY t.id ORDER BY t.criado_em DESC`
- Retornar `{ status: "ok", trilhas: [...] }`

**Função `adicionarEtapa`** — POST /api/trilhas/:id/etapas (professor):
- `parseInt(req.params.id, 10)` para `trilhaId`
- Validar `titulo`, `tipo`, `conteudo` no body → 400 se ausentes
- Validar tipo: `const TIPOS_VALIDOS = ["texto", "video", "link"]`; se `!TIPOS_VALIDOS.includes(tipo)` → 400 "Tipo inválido. Use: texto, video ou link."
- IDOR: `verificarDonoDaTrilha(trilhaId, req.usuario.id)` → 403 se falhar
- Calcular próxima ordem (NUNCA do body): `SELECT COALESCE(MAX(ordem), 0) + 1 AS proxima FROM trilha_etapas WHERE trilha_id = ?`
- `INSERT INTO trilha_etapas (trilha_id, ordem, titulo, tipo, conteudo) VALUES (?, ?, ?, ?, ?)`
- Capturar `ER_DUP_ENTRY` (race condition na UNIQUE KEY `etapa_unica`) → 409 "Conflito de posição. Tente novamente."
- Retornar `201` com `{ status: "ok", etapa: { id: resultado.insertId, trilha_id: trilhaId, ordem: proxima, titulo, tipo, conteudo } }`

**Função `minhas`** — GET /api/trilhas/minhas (aluno):
- Buscar turma do aluno: `SELECT turma_id FROM turma_alunos WHERE aluno_id = ? ORDER BY criado_em DESC LIMIT 1` com `[req.usuario.id]`
- Se aluno não está em turma → retornar `{ status: "ok", trilhas: [] }` (não é erro)
- Buscar trilhas com progresso calculado no backend via `SUM(IF())` (**não `COUNT(FILTER)` — MySQL não suporta**):
  ```sql
  SELECT
      t.id, t.titulo, t.disciplina, t.descricao,
      COUNT(te.id) AS total_etapas,
      SUM(IF(pe.concluido = 1, 1, 0)) AS etapas_concluidas,
      ROUND(SUM(IF(pe.concluido = 1, 1, 0)) / NULLIF(COUNT(te.id), 0) * 100) AS percentual
  FROM trilhas t
  LEFT JOIN trilha_etapas te ON te.trilha_id = t.id
  LEFT JOIN progresso_etapa pe ON pe.etapa_id = te.id AND pe.aluno_id = ?
  WHERE t.turma_id = ? AND t.ativa = 1
  GROUP BY t.id, t.titulo, t.disciplina, t.descricao
  ORDER BY t.criado_em DESC
  ```
  Parâmetros: `[req.usuario.id, turmaId]` — `req.usuario.id` aparece duas vezes (no LEFT JOIN da subquery e como parâmetro)
- Retornar `{ status: "ok", trilhas: [...] }`

**Função `getEtapas`** — GET /api/trilhas/:id/etapas (professor e aluno):
- `parseInt(req.params.id, 10)` para `trilhaId`
- Se `req.usuario.perfil === "aluno"`: verificar membership via JOIN `turma_alunos → trilhas`: `SELECT ta.id FROM turma_alunos ta JOIN trilhas t ON t.turma_id = ta.turma_id WHERE t.id = ? AND ta.aluno_id = ? LIMIT 1` → 403 se não for membro
- Se `req.usuario.perfil === "professor"`: `verificarDonoDaTrilha(trilhaId, req.usuario.id)` → 403 se não for dono
- `SELECT id, ordem, titulo, tipo, conteudo FROM trilha_etapas WHERE trilha_id = ? ORDER BY ordem ASC`
- Se aluno: enriquecer cada etapa com `concluido` (boolean): `SELECT etapa_id, concluido FROM progresso_etapa WHERE aluno_id = ? AND etapa_id IN (?)` → mapear por `etapa_id`; se `etapaIds.length === 0`, pular a query
- Retornar `{ status: "ok", etapas: [...] }` — para professor, `concluido` não é incluído

**Função `concluirEtapa`** — POST /api/trilhas/etapas/:etapaId/concluir (aluno):
- `parseInt(req.params.etapaId, 10)` para `etapaId`
- `const alunoId = req.usuario.id` — **NUNCA de `req.body`** (IDOR)
- IDOR + membership: verificar que o aluno é membro da turma da trilha desta etapa: `SELECT ta.id FROM turma_alunos ta JOIN trilhas t ON t.turma_id = ta.turma_id JOIN trilha_etapas te ON te.trilha_id = t.id WHERE te.id = ? AND ta.aluno_id = ? LIMIT 1` → 403 se não for membro
- `INSERT ... ON DUPLICATE KEY UPDATE` (idempotente para duplo-clique):
  ```sql
  INSERT INTO progresso_etapa (aluno_id, etapa_id, concluido, concluido_em)
  VALUES (?, ?, 1, CURRENT_TIMESTAMP)
  ON DUPLICATE KEY UPDATE
      concluido = 1,
      concluido_em = CURRENT_TIMESTAMP
  ```
  Parâmetros: `[alunoId, etapaId]`
- Retornar `{ status: "ok", message: "Etapa marcada como concluída." }`

**Função `getProgresso`** — GET /api/trilhas/:id/progresso (aluno):
- `parseInt(req.params.id, 10)` para `trilhaId`
- IDOR: `SELECT ta.id FROM turma_alunos ta JOIN trilhas t ON t.turma_id = ta.turma_id WHERE t.id = ? AND ta.aluno_id = ? LIMIT 1` → 403 se não for membro
- Query de progresso com `SUM(IF())` e `NULLIF` para evitar divisão por zero:
  ```sql
  SELECT
      COUNT(te.id) AS total_etapas,
      SUM(IF(pe.concluido = 1, 1, 0)) AS etapas_concluidas,
      ROUND(SUM(IF(pe.concluido = 1, 1, 0)) / NULLIF(COUNT(te.id), 0) * 100) AS percentual
  FROM trilha_etapas te
  LEFT JOIN progresso_etapa pe ON pe.etapa_id = te.id AND pe.aluno_id = ?
  WHERE te.trilha_id = ?
  ```
  Parâmetros: `[req.usuario.id, trilhaId]`
- Retornar: `{ status: "ok", progresso: { total_etapas: rows[0].total_etapas || 0, etapas_concluidas: rows[0].etapas_concluidas || 0, percentual: rows[0].percentual || 0 } }`

### Tarefa 2 — Criar `backend/routes/trilhas.js`; registrar em `backend/server.js`

**Criar `backend/routes/trilhas.js`:**

```js
const { Router } = require("express");
const { autenticar } = require("../middleware/auth");
const trilhasController = require("../controllers/trilhasController");

const roteador = Router();

// Guards inline — mesmo padrão de turmas.js; não extrair para arquivo compartilhado
function verificarProfessor(req, res, next) {
    if (req.usuario.perfil !== "professor") {
        return res.status(403).json({ status: "erro", message: "Apenas professores podem realizar esta ação." });
    }
    next();
}

function verificarAluno(req, res, next) {
    if (req.usuario.perfil !== "aluno") {
        return res.status(403).json({ status: "erro", message: "Apenas alunos podem realizar esta ação." });
    }
    next();
}

// ORDEM IMPORTA: rotas literais e segmentos específicos antes de :param
roteador.get("/minhas", autenticar, verificarAluno, trilhasController.minhas);
roteador.get("/turma/:turmaId", autenticar, verificarProfessor, trilhasController.listarPorTurma);
roteador.post("/etapas/:etapaId/concluir", autenticar, verificarAluno, trilhasController.concluirEtapa);
roteador.post("/", autenticar, verificarProfessor, trilhasController.criar);
roteador.post("/:id/etapas", autenticar, verificarProfessor, trilhasController.adicionarEtapa);
roteador.get("/:id/etapas", autenticar, trilhasController.getEtapas);
roteador.get("/:id/progresso", autenticar, verificarAluno, trilhasController.getProgresso);

module.exports = roteador;
```

## Threat Model

| Ameaça STRIDE | Componente | Disposição | Mitigação |
|---------------|------------|------------|-----------|
| Tampering / IDOR | Professor A adiciona etapa à trilha do Professor B | mitigar | `verificarDonoDaTrilha(trilhaId, req.usuario.id)` antes de todo POST em /:id/etapas; 403 se professor_id não corresponde |
| Tampering / IDOR | Aluno marca progresso com `aluno_id` de outro no body | mitigar | `concluirEtapa` usa exclusivamente `req.usuario.id` (token JWT) como `aluno_id`; `req.body` não é lido para IDs de usuário |
| Tampering / IDOR | Aluno marca etapa de trilha de turma alheia | mitigar | `concluirEtapa` verifica membership via JOIN `turma_alunos → trilhas → trilha_etapas` antes do INSERT; 403 se não membro |
| Information Disclosure | Aluno lê etapas de trilha de turma que não é membro | mitigar | `getEtapas` para aluno verifica membership via JOIN antes do SELECT de etapas |
| Elevation of Privilege | Aluno cria trilha ou adiciona etapa | mitigar | Guards `verificarProfessor` em POST /api/trilhas e POST /api/trilhas/:id/etapas |
| Tampering | Race condition em `adicionarEtapa` viola UNIQUE KEY `etapa_unica` | aceitar | Capturado como `ER_DUP_ENTRY` → 409; usuário pode tentar novamente; risco é muito baixo em uso TCC |
| Denial of Service | Acumulação de registros em `progresso_etapa` por spam | aceitar | `ON DUPLICATE KEY UPDATE` é idempotente; spam apenas atualiza `concluido_em`, sem crescimento indefinido |
